8.What is creation phase and execution phase?


        Compiler generates a global execution context before executing the code.
        inside this we have two phases creation phase and execution phase.

        Creation phase: 
        During creation phase global execution context ,allowcates  memory space for functions and variables .
        functions will be stored as with whole function body and variables are declared and assigned default value of undefined.
        -It is also known as memory allowcation phase.

        Execution phase:
        During the execution phase javascript executes our code line by line and assign the values to the variables.
        -Then function are executed.
        -It  is also known as code execution phase.

10.What are function constructors?


        IN JS, A constructor is a special function that creates  and initilazes instance of a object.
        A constructor get called when an object is created using new keyword.
        -We have multiple constructors , function constructor for functions , object constructors for object , array constructors for arrays.
        -When constructor get invoked in js , this operations are perform:
                -new empty object gets invoked.
                -this keyword refers to the new object and it becomes the current instance object.
                -new object returned as the return value of constructor.

        EX:
        function user1(name,id){
            this.name = name;
            this.id = id;
        }
        let user = new user1('abc', 10001)

11.What are callbacks in js?


        In JS, A callback function is a function passed into another function as an argument.
        This function invokes inside the outer function to complete an action.

        EX:
        function myfun1(name){
            console.log('hello',name)
        }
        function myfun2(callback){
            let lang = 'javascript';
            callback(lang)
        }
        myfun2(myfun1);

12.Explain prototypes?


        In JS, prototypes are the mechanism by javascript object inherit features from one another.
        Every object in javascript has a bulit-in prototype , which is called its prototype.
        -The prototype is itself an object , so the prototype will have its own prototype , making what's called prototype chain.
        -The chain ends , when we reach a prototype that has null for its own prototype.
        -It is used for allow us to add new properties to object constructor.

        EX:
        function person(name,age){
            this.name = name;
            this.age = age;
        }
        person.prototype.state = 'ts';
        let p = new person('abc', 23);
        console.log(p.state);
